DICOM Terms and Abbreviations

DICOM Terms and Abbreviations

SC

SC: Secondary Capture, object intended for images acquired by capturing video and digitizing it again. Also used for Digitized film, and any other modality which has (yet) no own IOD such as 3D generated images.

SCP

SCP: Service Class Provider, DICOM AE which functions as a server, or "provides" a service such as Storage, Print etc.

SCSI

SCSI: Small Computer System Interface.

SCU

SCU: Service Class User, DICOM AE which functions as a client, or "uses" a service, i.e. for printing, storage, etc.

SDL

SDL: Source Density LUT.

SH

SH: Abbreviation for a Value Representation: Short String. For details see here.

SL

SL: Abbreviation for a Value Representation: Signed Long. For details see here.

SLIP

SLIP: Single Line Internet Protocol.

SOP

SOP: Service Object Pair.

SOP Class

SOP Class: Service Object Pair, combination of a service such as Store, Retrieve, and an Object such as Image, Print queue, etc. The Class defines the rules on how to use these; a Class is negotiated as a capability between devices during negotiation of the Association. Analogy of a Class is a type of fruit, e.g. Granny Smith.

SOP Instance

SOP Instance: This is a single, unique occurrence of a SOP Class. It is always identified with a SOP Instance UID). An analogy is a specific apple of the type ("Class") Granny Smith.

SQ

SQ: Abbreviation for a Value Representation: Sequence of Items. For details see here.

SS

SS: Abbreviation for a Value Representation: Signed Short. For details see here.

ST

ST: Abbreviation for a Value Representation: Short Text. For details see here.

ST: Abbreviation for a type of modality: Single-Photon Emission Computed Tomography (SPECT).

Service

Service: Entity that allows layer-to-layer communication

Service Class

Service Class: A service Class is a group of one or more SOP Classes. E.g. the Storage Service class contains all of the Storage SOP Classes (CT-Store, MR-Store, etc.).
A structured description of a service supported by cooperating DICOM application entities using specific DICOM commands and acting on a specific class of information object.

Service object pair

Service object pair: The combination of a service and the type of information object upon which it can operate. (SOP).

Service provider

Service provider: The sum of the wire, plus all of the lower levels (below the application level).

Standard Data Element

Standard Data Element: A Data Element defined in the DICOM standard, and therefore listed in the DICOM Data Element Dictionary. See also Private Data Element, Retired Data Element.

Study

Study or Exam: A collection (set) consisting of series of images.

Supplement

Supplement: A document that extends the DICOM standard. Information in an approved Supplement has the status of "standard", but is not yet incorporated into the proper places in the standard.
