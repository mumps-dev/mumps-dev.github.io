5.4.40 VALUE wattribute

Definition

For a build-mode window VALUE and its descendant nodes specify the currently selected gadget(s). VALUE specifies the number of gadgets selected, and selections are indicated by descendant nodes of the form

^$W[INDOW] ( wname , vkeyword , gname )

where

    vkeyword::= expr V VALUE

and gname identifies a selected gadget.

If an application attempts to create a descendant node of VALUE which does not identify a gadget which is a child of the window, an error condition occurs with $ECODE containing ",M48," and $EREF containing a value that indicates the ssvn reference for which the error occurred.

If an application kills the descendant node of VALUE which identifies the principal gadget, the PSELECT wattribute is reset to be the first remaining descendant node of VALUE, if any, otherwise becoming undefined.

If an application kills the VALUE wattribute it is reset to its default and the PSELECT wattribute is also implicitly killed.

For a window that is not in build mode, the wattribute and its descendant nodes may be set and referenced but are ignored.

Domain VALUE has a domain of intexpr and descendant nodes have a domain of emptystring.
Access Create, Modify, Reference
Default 0
