5.4.16 Revextorder

(Operation type 73) The revextorder operation fetches extended information about the global reference preceding the global reference given in its global reference field <LS>.

<LS>
<LI> Global Reference

Figure 35 – Revextorder request

The revextorder response shall include the preceding subscript on the same level as the final subscript of the requested global reference, or the preceding global name; the $data value of the preceding global reference; and its value, if any.

The fields of the extorder response and their sequence shall be:

  1. Order value: <SS> the preceding subscript or the preceding global name as specified in 5.4.11.
  2. $data value: <SI> the $data value of the global reference defined by order value. ANSI/MDC X11.1, 7.1.5.3, defines the $data values that indicate the presence of a global value and descendants.
  3. Global value: <LS> the value of the global variable defined by order value. If the $data value indicates that no value exists, this field shall equal the empty string.
